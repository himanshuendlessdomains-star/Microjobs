# BountyHive Smart Contracts

## BountyEscrow.tact

One contract instance per bounty. It holds the creator's pool amount and
automatically distributes rewards to winners after the deadline.

### Prerequisites

```bash
# From the project root:
npm install --save-dev @ton/blueprint @ton/core @ton/ton @ton/crypto
```

### Compile

```bash
npx blueprint build BountyEscrow
```

This outputs `build/BountyEscrow.compiled.json` containing the code cell BOC.

### Deploy to testnet

1. Create a deploy script:

```bash
npx blueprint run deployBountyEscrow --testnet
```

2. Or use the Blueprint interactive deployer:

```bash
npx blueprint
```

### Deploy to mainnet

Same as testnet but use `--mainnet` flag. **Never deploy untested contracts to
mainnet.** Run thorough tests on testnet first.

### After deployment

The contract address is unique **per bounty** because it is derived from the
init parameters (creator address + winner count + deadline timestamp).

The frontend needs the **compiled code cell** (the `hex` field from
`build/BountyEscrow.compiled.json`) to compute each bounty's address
deterministically before deploying.

### Adding the code cell to the frontend

1. After `npx blueprint build`, copy the hex string from
   `build/BountyEscrow.compiled.json`.

2. Create `src/lib/escrow-code.ts`:

```typescript
// Generated by: npx blueprint build BountyEscrow
export const BOUNTY_ESCROW_CODE_HEX = "b5ee9c72..."; // paste full hex here
```

3. In the frontend, use `@ton/core` to compute the address:

```typescript
import { Cell, Address, StateInit, contractAddress } from "@ton/core";
import { BOUNTY_ESCROW_CODE_HEX } from "@/lib/escrow-code";

function computeEscrowAddress(
  creator: Address,
  winnerCount: number,
  deadline: number,   // Unix seconds
): Address {
  const code = Cell.fromHex(BOUNTY_ESCROW_CODE_HEX);

  // Must match the init() parameter order in BountyEscrow.tact
  const data = beginCell()
    .storeAddress(creator)
    .storeUint(winnerCount, 8)
    .storeUint(deadline, 48)
    .storeCoins(0)           // poolAmount
    .storeBit(false)         // settled
    .storeStringTail("")     // bountyId
    .endCell();

  const stateInit: StateInit = { code, data };
  return contractAddress(0, stateInit);   // workchain 0
}
```

4. Use this address as `NEXT_PUBLIC_ESCROW_ADDRESS` — or better, pass the
   unique address per bounty into the TonConnect send transaction call.

### MVP Shortcut (testnet only)

While contracts are not yet deployed, point `NEXT_PUBLIC_ESCROW_ADDRESS` in
`.env.local` to any TON testnet wallet you control. All pool amounts and entry
fees will go there. You can distribute rewards manually from that wallet.

Once the contracts are compiled and deployed, replace the address with the
per-bounty contract address computed above.
